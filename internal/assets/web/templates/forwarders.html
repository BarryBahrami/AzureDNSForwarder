Forwarders

Exact or wildcard forward zones. Prefix a name with *. (for example, *.corp.local) to make it a wildcard that matches the zone and every subdomain. Server list is tried in order.

Check "Send to Azure DNS" to forward this zone to 168.63.129.16, "Send to AWS DNS" to forward to 169.254.169.253, or "Send to GCP DNS" to forward to 169.254.169.254. The upstreams field will be ignored when a cloud DNS checkbox is selected. Otherwise enter one or more comma-separated nameserver IPs. "Least Latency Response" is only available for exact (non-wildcard) names; when checked the system periodically pings every returned record and answers with the lowest-latency targets. "Do not sync" keeps the zone local; leave unchecked and set peer names to sync only to specific partners.

Default upstreams (catch-all)

These receive any query not matched by a forward zone above. Toggle each upstream on or off; you can also remove the Azure DNS upstream entirely if you only want specific zones to forward.

Check "Use Azure DNS" to add 168.63.129.16, "Use AWS DNS" to add 169.254.169.253, or "Use GCP DNS" to add 169.254.169.254 as your catch-all. The address and port fields will be ignored when a cloud DNS checkbox is selected. "Do not sync" keeps the upstream local; leave unchecked and set peer names to sync only to specific partners.
